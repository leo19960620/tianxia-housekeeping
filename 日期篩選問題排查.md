# 租借紀錄日期篩選問題排查

## 問題描述
租借紀錄頁籤選擇今天的日期後，沒有顯示今天的記錄。

## 可能原因

### 1. 時區轉換問題
後端使用台北時區（Asia/Taipei）進行日期比較，但資料可能以 UTC 儲存。

### 2. 資料庫時間戳記格式
PostgreSQL 的 `timestamp` 欄位可能沒有時區資訊。

## 排查步驟

### 步驟 1：檢查資料庫中的實際時間

在 Supabase SQL Editor 執行：

```sql
-- 查看最近的租借記錄和時間戳記
SELECT 
    id,
    bicycle_id,
    rental_start_time,
    rental_start_time AT TIME ZONE 'UTC' AT TIME ZONE 'Asia/Taipei' as taipei_time,
    DATE(rental_start_time AT TIME ZONE 'UTC' AT TIME ZONE 'Asia/Taipei') as taipei_date
FROM bicycle_rentals
ORDER BY rental_start_time DESC
LIMIT 10;
```

### 步驟 2：測試日期篩選

```sql
-- 測試今天的日期篩選（手動替換日期）
SELECT 
    id,
    bicycle_id,
    rental_start_time,
    DATE(rental_start_time AT TIME ZONE 'UTC' AT TIME ZONE 'Asia/Taipei') as taipei_date
FROM bicycle_rentals
WHERE DATE(rental_start_time AT TIME ZONE 'UTC' AT TIME ZONE 'Asia/Taipei') = '2026-01-20'
ORDER BY rental_start_time DESC;
```

### 步驟 3：檢查時區設定

```sql
-- 查看資料庫時區設定
SHOW timezone;

-- 查看當前時間
SELECT 
    NOW() as utc_now,
    NOW() AT TIME ZONE 'Asia/Taipei' as taipei_now,
    CURRENT_DATE as current_date;
```

## 可能的解決方案

### 方案 1：簡化時區處理（推薦）

如果資料都是在台灣建立的，可以簡化 SQL：

```sql
-- 直接使用 DATE 函數（假設資料庫時區已設定為台北）
WHERE DATE(rental_start_time) = $1
```

修改後端：
```javascript
// 在 bicycle-rentals.js 和 umbrella-rentals.js
if (date) {
    conditions.push(`DATE(rental_start_time) = $${paramCount++}`);
    params.push(date);
}
```

### 方案 2：確保資料庫時區正確

在 Supabase 設定資料庫時區為 Asia/Taipei：

```sql
ALTER DATABASE postgres SET timezone TO 'Asia/Taipei';
```

### 方案 3：前端加上除錯資訊

暫時在前端顯示 API 請求參數，確認傳送的日期是否正確：

```javascript
console.log('查詢日期:', selectedDate);
console.log('API 參數:', { date: selectedDate });
```
