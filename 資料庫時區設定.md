# 資料庫時區設定指南

## 問題說明

**狀況**：資料庫（Supabase PostgreSQL）存的時間都是 UTC+0，但我們需要顯示台北時間（UTC+8）。

**解決方案**：有兩種方式

---

## 方案 1：在 Supabase 設定資料庫時區（推薦）

### 步驟 1：連接到 Supabase SQL Editor

1. 前往 Supabase Dashboard
2. 選擇專案
3. 點擊左側選單的 **SQL Editor**

### 步驟 2：設定資料庫時區

執行以下 SQL：

```sql
-- 設定資料庫時區為台北（UTC+8）
ALTER DATABASE postgres SET timezone TO 'Asia/Taipei';

-- 檢查設定是否生效
SHOW timezone;
```

**預期輸出**：
```
 timezone     
-------------
 Asia/Taipei
```

### 步驟 3：重新連線

設定生效後，新的連線會自動使用 Asia/Taipei 時區。

**優點**：
- ✅ 一次設定，全系統生效
- ✅ 所有 `NOW()`、`CURRENT_TIMESTAMP` 都會自動使用台北時間
- ✅ 前端不需要轉換時區

**缺點**：
- ⚠️ 已存在的資料不會自動轉換

---

## 方案 2：在前端轉換時區（目前使用）

### 實作方式

在前端顯示時間時，將 UTC+0 轉為 UTC+8：

```javascript
const formatTime = (dateString) => {
    if (!dateString) return '';
    // 資料庫存 UTC+0，轉換為台北時間（UTC+8）
    const date = new Date(dateString);
    const taipeiTime = new Date(date.getTime() + (8 * 60 * 60 * 1000));
    const hours = taipeiTime.getUTCHours().toString().padStart(2, '0');
    const minutes = taipeiTime.getUTCMinutes().toString().padStart(2, '0');
    return `${hours}:${minutes}`;
};
```

**優點**：
- ✅ 不需要修改資料庫設定
- ✅ 已存在的資料不受影響

**缺點**：
- ❌ 每個顯示時間的地方都要轉換
- ❌ 容易遺漏或出錯

## 推薦做法

### 使用方案 1（設定資料庫時區）

1. **執行 SQL 設定時區**
2. **前端簡化時間顯示**
   ```javascript
   const formatTime = (dateString) => {
       if (!dateString) return '';
       const date = new Date(dateString);
       return date.toLocaleTimeString('zh-TW', { 
           hour: '2-digit', 
           minute: '2-digit',
           hour12: false 
       });
   };
   ```

### 後續新資料

設定時區後，所有新建立的記錄都會自動使用台北時間：

```sql
-- 建立新記錄時，自動使用台北時間
INSERT INTO bicycle_rentals (bicycle_id, rented_by)
VALUES (1, 'John');

-- created_at 會自動設為當前台北時間
```

## 驗證方式

### 檢查當前時間

```sql
SELECT 
    NOW() as current_time,
    NOW() AT TIME ZONE 'UTC' as utc_time,
    NOW() AT TIME ZONE 'Asia/Taipei' as taipei_time;
```

### 檢查已存在的資料

```sql
SELECT 
    id,
    created_at,
    created_at AT TIME ZONE 'UTC' AT TIME ZONE 'Asia/Taipei' as taipei_time
FROM bicycle_rentals
ORDER BY created_at DESC
LIMIT 5;
```

## 常見問題

### Q1: 設定時區後，舊資料會受影響嗎？

**A**: 不會。舊資料保持原樣（UTC+0），但查詢時可以用 `AT TIME ZONE` 轉換：

```sql
-- 顯示時轉換為台北時間
SELECT created_at AT TIME ZONE 'Asia/Taipei' FROM bicycle_rentals;
```

### Q2: 前端還需要轉換時區嗎？

**A**: 設定資料庫時區後：
- **新資料**：不需要轉換（已經是台北時間）
- **舊資料**：需要在查詢時轉換，或在前端轉換

### Q3: 如何統一處理所有時間？

**A**: 最好的做法是：

1. **設定資料庫時區為 Asia/Taipei**
2. **更新所有時間欄位為台北時間**
   ```sql
   -- 更新舊資料（一次性）
   UPDATE handovers 
   SET created_at = created_at AT TIME ZONE 'UTC' AT TIME ZONE 'Asia/Taipei';
   ```
3. **前端直接使用時間，不再轉換**
