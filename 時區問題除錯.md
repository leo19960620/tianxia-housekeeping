# 時區問題除錯指南

## 問題描述
交接詳情右上角時間顯示錯誤（例如顯示 16:52 而非 08:52）

## 除錯步驟

### 步驟 1: 檢查後端返回的時間格式

在 App 的 `HandoverFormScreen.js` 檔案中，暫時加入 console.log：

```javascript
// 在 loadHandoverData 函數中，獲取資料後
if (detailRes.success) {
    console.log('原始 timestamp:', detailRes.data.timestamp);
    console.log('Date 物件:', new Date(detailRes.data.timestamp));
    console.log('getHours:', new Date(detailRes.data.timestamp).getHours());
    console.log('getUTCHours:', new Date(detailRes.data.timestamp).getUTCHours());
}
```

### 步驟 2: 分析輸出

可能的情況：

#### 情況 A: 後端返回帶 Z 的 ISO 格式
```
原始 timestamp: "2026-01-21T08:52:07.81Z"
```
- `Z` 表示 UTC 時間
- `getHours()` 會轉換為本地時區（UTC+8）→ 16:52 ❌
- `getUTCHours()` 會保持 UTC 時間 → 08:52 ✅

**解決方案**: 使用 `getUTCHours()`

#### 情況 B: 後端返回不帶時區的格式
```
原始 timestamp: "2026-01-21T08:52:07.81"
```
- 沒有時區資訊
- `getHours()` 會直接使用該時間 → 08:52 ✅
- `getUTCHours()` 會誤判為本地時間並轉換 → 00:52 ❌

**解決方案**: 使用 `getHours()`

#### 情況 C: 後端返回帶時區偏移的格式
```
原始 timestamp: "2026-01-21T08:52:07.81+08:00"
```
- `+08:00` 表示 UTC+8 時間
- `getHours()` 會轉換 → 可能錯誤
- 需要特殊處理

## 目前實作

已改用 `getUTC*` 方法：

```javascript
const date = new Date(handover.timestamp);
const hours = String(date.getUTCHours()).padStart(2, '0');
const minutes = String(date.getUTCMinutes()).padStart(2, '0');
```

**假設**: 後端返回的是 `2026-01-21T08:52:07.81Z` 格式（UTC 時間）

## 驗證方式

### 在瀏覽器 Console 測試

```javascript
// 測試不同格式
const utcFormat = new Date("2026-01-21T08:52:07.81Z");
console.log('UTC format getUTCHours:', utcFormat.getUTCHours()); // 8
console.log('UTC format getHours:', utcFormat.getHours()); // 16 (如果本地是 UTC+8)

const noTzFormat = new Date("2026-01-21T08:52:07.81");
console.log('No TZ getUTCHours:', noTzFormat.getUTCHours()); // 0 (錯誤)
console.log('No TZ getHours:', noTzFormat.getHours()); // 8 (正確)

const withTzFormat = new Date("2026-01-21T08:52:07.81+08:00");
console.log('With TZ getUTCHours:', withTzFormat.getUTCHours()); // 0
console.log('With TZ getHours:', withTzFormat.getHours()); // 8
```

## 最佳解決方案

**方案 1: 確保後端返回一致的格式**

在後端查詢時明確指定時區：

```javascript
// 在 db.js 的 query 函數中
await pool.query("SET timezone = 'Asia/Taipei'");
```

然後在返回時轉換為 ISO 字串：

```javascript
// 在 handover API 中
const result = await query('SELECT * FROM handovers WHERE id = $1', [id]);
result.rows[0].timestamp = new Date(result.rows[0].timestamp).toISOString();
```

**方案 2: 前端智能判斷**

檢查字串格式並選擇正確的方法：

```javascript
const formatTimestamp = (timestamp) => {
    const date = new Date(timestamp);
    // 檢查是否為 UTC 格式（以 Z 結尾）
    const isUTC = timestamp.endsWith('Z');
    
    const hours = isUTC 
        ? String(date.getUTCHours()).padStart(2, '0')
        : String(date.getHours()).padStart(2, '0');
    const minutes = isUTC
        ? String(date.getUTCMinutes()).padStart(2, '0')
        : String(date.getMinutes()).padStart(2, '0');
    
    return `${hours}:${minutes}`;
};
```
