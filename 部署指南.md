## 🚀 初次部署

### 第一步：資料庫部署 (Supabase)

#### 1. 建立 Supabase 專案
1. 前往 [Supabase](https://supabase.com)
2. 建立新專案
3. 選擇 `ap-northeast-1` (Tokyo) 區域
4. 設定資料庫密碼並儲存

#### 2. 執行 Migration
在 Supabase SQL Editor 中執行所有 migration：

```bash
# 本機準備
cd /Users/leo/tianxia-housekeeping/backend/database/migrations

# 按順序執行以下檔案內容到 Supabase SQL Editor：
# 1. init_schema.sql
# 2. add_rental_management.sql
# ... (其他 migration 檔案)
```

#### 3. 匯入資料（如有舊資料）
```bash
cd /Users/leo/tianxia-housekeeping/backend
./scripts/export-database.sh  # 先匯出本地資料
# 然後將資料匯入到 Supabase
```

#### 4. 取得連線資訊
前往 Supabase → Settings → Database，複製：
- Host (Pooler)
- Port
- Database name
- User
- Password

### 第二步：後端部署 (Render)

#### 1. 準備 GitHub Repository
```bash
cd /Users/leo/tianxia-housekeeping
git add .
git commit -m "準備部署到 Render"
git push origin main
```

#### 2. 建立 Render Web Service
1. 前往 [Render Dashboard](https://dashboard.render.com/)
2. 點擊 **New +** → **Web Service**
3. 連結 GitHub repository: `leo19960620/tianxia-housekeeping`

#### 3. 設定參數
- **Name**: `tianxia-housekeeping-api`
- **Region**: `Singapore (Southeast Asia)`
- **Branch**: `main`
- **Root Directory**: `backend`
- **Runtime**: `Node`
- **Build Command**: `npm install`
- **Start Command**: `npm start`
- **Instance Type**: `Starter` (USD 7/月，推薦) 或 `Free`

#### 4. 設定環境變數
在 Render Environment 區塊新增：

```env
DB_HOST=aws-1-ap-northeast-1.pooler.supabase.com
DB_PORT=5432
DB_USER=postgres.yyjcuhacwlwvhesvzhak
DB_PASSWORD=你的資料庫密碼
DB_NAME=postgres
NODE_ENV=production
PORT=3001
JWT_SECRET=tianxia_housekeeping_secret_key_2026
JWT_EXPIRES_IN=7d
```

#### 5. 部署並測試
- 等待部署完成（約 2-5 分鐘）
- 取得後端網址：`https://tianxia-housekeeping-api.onrender.com`
- 測試健康檢查：`https://tianxia-housekeeping-api.onrender.com/health`

#### 4. 驗證部署

### 第四步：Android APK 部署

#### 1. 打包 APK
```bash
cd /Users/leo/tianxia-housekeeping/TianxiaHousekeeping

# 更新版本號（編輯 android/app/build.gradle）
# versionCode 加 1
# versionName 更新為新版本號（例如：1.2.0）

# 打包
cd android
./gradlew assembleRelease
```

#### 2. 上傳 APK 到 Web
```bash
# APK 位置：android/app/build/outputs/apk/release/app-release.apk
# 將 APK 複製到 public 資料夾，並命名為 app-release.apk (與 download.html 連結一致)
cp app/build/outputs/apk/release/app-release.apk ../../public/app-release.apk
```

#### 3. 更新下載頁面資訊
編輯 `public/download.html`：
- 更新 `最新版本` 號碼
- 更新 `更新日期`
- 若需要，更新 `download` 屬性中的檔名 (例如：`download="天下房務管理_v1.2.apk"`)

#### 4. 重新部署 Web（包含新 APK）
```bash
cd /Users/leo/tianxia-housekeeping
npm run build
npx wrangler pages deploy dist --project-name=tianxia-housekeeping
```

#### 5. 下載連結
APK 下載網址：`https://tianxia-housekeeping.pages.dev/download` (進入下載頁面)
直接下載連結：`https://tianxia-housekeeping.pages.dev/app-release.apk`

   - 自動重新建置並部署

### 更新 Web 前端

1. **建置並部署**
   ```bash

   npx wrangler pages deploy dist --project-name=tianxia-housekeeping
   ```

2. **驗證**
   - 清除瀏覽器快取
   - 重新載入頁面測試

### 更新 Android App

1. **更新版本號**
   ```bash
   # 編輯 android/app/build.gradle
   # versionCode: 遞增
   # versionName: 更新為新版本
   ```

2. **重新打包與部署**
   ```bash
   cd TianxiaHousekeeping/android
   ./gradlew assembleRelease
   
   # ⚠️ 重要：請務必將 APK 複製到 public/ 資料夾，而不是 dist/
   cp app/build/outputs/apk/release/app-release.apk ../../public/app-release.apk
   
   # (必做) 編輯 public/download.html 更新版本號與日期
   # 注意：請勿編輯 dist/download.html，因為它會在 build 時被覆蓋
   
   # 重新部署 Web（包含新 APK）
   cd ../..
   npm run build
   npx wrangler pages deploy dist --project-name=tianxia-housekeeping
   ```

3. **通知使用者更新**
   - 使用者前往下載頁面
   - 下載並安裝新版 APK

### 更新 iOS App

1. 連接 iPhone 到 Mac
2. 開啟 `TianxiaHousekeeping/ios/TianxiaHousekeeping.xcworkspace`

4. 按 `Cmd + R` 編譯並安裝

## ⚠️ 常見問題

### 1. Render 服務休眠（Free 方案）
**問題**：免費方案在無活動時會休眠，首次請求需等待 30-60 秒啟動

**解決方案**：
- 升級到 Starter 方案 (USD 7/月)
- 或使用定時 ping 保持活躍：
  ```bash
  # 使用 UptimeRobot 或 cron-job.org
  # 每 5 分鐘 ping 一次
  curl https://tianxia-housekeeping-api.onrender.com/health
  ```

### 2. Cloudflare Pages 快取問題
**問題**：更新後前端仍顯示舊版本

**解決方案**：
```bash
# 清除瀏覽器快取
# 或在 Cloudflare Dashboard 清除快取
# 或在網址後加 ?v=時間戳
```

### 3. 下載頁面一直變回舊版
**問題**：修改了 download.html 但部署後又變回舊的內容

**原因**：
- 您可能直接修改了 `dist/download.html`
- `npm run build` 會將 `public/` 的內容覆蓋到 `dist/`

**解決方案**：
- ✅ **請務必修改 `public/download.html`**
- 修改後再執行 `npm run build`
- 這樣正確的內容才會被打包進 `dist/` 進行部署

### 4. APK 無法安裝
**問題**：Android 手機提示無法安裝

**解決方案**：
- 解除安裝舊版本
- 允許安裝來自未知來源的應用程式
- 檢查 APK 是否完整下載

### 4. API 連線失敗
**問題**：App 或 Web 無法連接後端

**檢查清單**：
- ✅ Render 服務是否正常運行
- ✅ Supabase 資料庫是否可連線
- ✅ 環境變數是否正確設定
- ✅ CORS 設定是否正確
